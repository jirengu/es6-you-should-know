let 和 const

let 和 const

let

1. 使用 let 替换 var

let name = 'Kevin'
for(let i = 0; i < 10; i++) {
  console.log(i)
}

2. let不可声明前置

a = 3  // Uncaught ReferenceError: a is not defined
let a

3. let不可重复声明

let a = 3
let a = 4  // Uncaught SyntaxError: Identifier 'a' has already been declared

4. 存在块级作用域

for(let i = 0; i < 3; i++){
  console.log(i)
}
console.log(i) // Uncaught ReferenceError: a is not defined


if(true){
  let a = 1
}
console.log(a)  // Uncaught ReferenceError: a is not defined

IIFE的替换

(function(){
  var a = 1
}())

{
  let a = 1
}

一个典型的范例

for(var i = 1; i < 4; i++){
  setTimeout(function(){
    console.log(i)   
  }, 1000*i)
}

以上代码会在1秒后输出 4、2秒后输出 4、3秒后后输出 4。如果想实现1秒后输出 1、2秒后输出 2、3秒后后输出 3，则可以使用下面的方法

for(var i = 1; i < 4; i++){
  (function(j){
    setTimeout(function(){
      console.log(j)   
    }, 1000*j)
  })(i)
}

或者可以使用let：

for(let i = 1; i < 4; i++){
  setTimeout(function(){
    console.log(i)   
  }, 1000*i)
}

在代码块内，使用let命令声明变量之前，该变量都是不可用的。这在语法上，称为“暂时性死区”（temporal dead zone，简称 TDZ）

const

1. 使用 const 替换 var

const URL = 'https://xiedaimala.com'
const ERR_MSG = '啊哈，报错了'

2. const 声明的常量不可改变

const a = 1
a = 2 //Uncaught TypeError: Assignment to constant variable

以上代码中 a 初始化为1后，再次设置为2时，就会报错。

const obj = {a: 1}
obj.a = 2   //Ok
obj.b = 3   //Ok
obj = {a: 2}  //Uncaught TypeError: Assignment to constant variable

以上代码修改 obj 的属性并没有报错。可以这样理解：obj 存储的是对象的地址，给对象属性赋值时并没有修改存储的这个地址，所以不报错。 但obj = {a: 2}相当于更换 obj 里存储的地址，所以报错。

3. 适用于let的特性同样适用于const

  • 不可声明前置
  • 不可重复声明
  • 存在块级作用域
